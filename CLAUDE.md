# AgentHub Starter — Claude Code Context

## What this repo is

A starter agentic application demonstrating AG-UI protocol + LangGraph + AWS Bedrock with a chat-plus-canvas UI and declarative widgets. Reference use case: CloudWatch Logs Insights query agent with HITL approval. See PLAN.md for the full build plan.

## Repo layout

Single project, not a monorepo. Next.js app at the root, Python agent in `agent/`. No workspaces. Run both with `npm run dev`.

## Critical conventions

- **Widget schema source of truth is `agent/agent/widgets.py`.** TypeScript types in `lib/widgets.ts` mirror it by hand. If you change one, change the other in the same commit.
- **Bedrock model IDs use cross-region inference profiles** (e.g. `us.anthropic.claude-sonnet-4-5-...`), read from `BEDROCK_AGENT_MODEL` env var.
- **AWS auth via environment variables only.** Don't depend on `~/.aws/config` or credential helpers.
- **Tenant scoping is via AssumeRole.** The agent's CloudWatch client is constructed per-thread using the tenant's IAM role, cached in memory by `(tenant_id, thread_id)`.
- **The LLM never formats AG-UI events directly.** It calls `create_widget` / `update_widget` / `remove_widget` tools. The LangGraph node implementing those tools translates to AG-UI custom events.
- **Widget IDs are ULIDs**, generated by the agent.
- **Widget updates use JSON Patch** (RFC 6902, via `fast-json-patch` on frontend, `jsonpatch` on Python).
- **Checkpointing is in-memory (`MemorySaver`) for the starter.** Postgres is Phase 7. Don't add Postgres dependencies until then.

## Stack quick reference

- Frontend: Next.js 15 App Router, React 19, TS strict, Tailwind v4, shadcn/ui (New York, Slate), `@ag-ui/client`
- Backend: Python 3.12, FastAPI, LangGraph + MemorySaver, `langchain-aws` (ChatBedrockConverse), `ag-ui-protocol`, `sse-starlette`
- Package managers: npm (frontend), uv (Python)
- No Docker, no Postgres until Phase 7

## Architecture in one paragraph

User types in chat → Next.js Client Component → Route Handler verifies JWT and proxies SSE to FastAPI → LangGraph run starts, emits AG-UI events including widget_create/update/remove → events stream back through the proxy → `@ag-ui/client` parses → `useAgent` hook reducer updates state → React renders chat transcript on the left and canvas widget stack on the right. User actions on widgets POST to `/api/agent/widget-action`, which resumes the LangGraph thread via interrupt response.

## Code style

- TypeScript: strict mode, no `any`, prefer discriminated unions over flags. Functional components, hooks for state.
- Python: type hints required, mypy strict, Pydantic for all external interfaces, structlog for logging (no print).
- Shadcn components in `components/ui/` are owned code — edit them when needed.
- Custom non-chat components: `components/app/`
- Chat components: `app/(app)/chat/_components/`

## Things to ask before doing

- Changing the widget schema (cascades through both apps)
- Adding a new widget type beyond the planned six
- Adding a dependency not in PLAN.md's stack
- Touching auth or tenant scoping logic
- Switching from MemorySaver to PostgresSaver before Phase 7

## Things to just do

- Implementing planned widgets and components
- Adding tests
- Improving styling within established design tokens
- Refactoring within a phase's scope
- Updating docs

## Workflow

- One phase at a time, in order
- Commit at each working milestone
- Lint/typecheck before commits: `npm run lint && npm run typecheck`, `cd agent && uv run ruff check && uv run mypy`
- Update `PROGRESS.md` after each phase

## Useful commands

```bash
# Start everything
npm run dev                                # both apps via concurrently

# Frontend only
npm run dev:web

# Agent only
npm run dev:agent
# or directly:
cd agent && uv run uvicorn agent.main:app --reload --port 8000

# Type and lint
npm run lint && npm run typecheck
cd agent && uv run ruff check && uv run mypy

# Add shadcn component
npx shadcn@latest add <component>

# Add Python dep
cd agent && uv add <package>
```
